# nonestore 如何在 React、Vue、Angular 中使用与适配

## 1. 先给结论

`nonestore` 目前可以分成两层理解：

1. 核心层：`create`
2. React 适配层：`createStore`、`useStore`

所以它的跨框架适用性可以直接总结成一句话：

> `create` 可以跨框架使用，`createStore` 和 `useStore` 只能在 React 中使用。

也就是说：

- 在 React 中：可以完整使用
- 在 Vue 中：可以使用核心层，但要自己做适配
- 在 Angular 中：可以使用核心层，但也要自己做适配

因此 `nonestore` 现在并不是“只支持 React”，而是：

- **核心是跨框架的**
- **现成绑定目前只有 React**

---

## 2. 为什么会这样

因为这个库的结构本身就是两层。

### 2.1 核心层

核心层提供的是状态容器能力：

- `getState()`
- `setState()`
- `subscribe()`
- `actions`
- `computed`

这些能力本身不依赖 React、Vue、Angular 中任何一个框架。

它只是一个“状态仓库”。

所以核心层天然可以跨框架。

### 2.2 React 绑定层

`createStore` 和 `useStore` 属于 React 绑定。

例如：

```tsx
const [useCounter, counterStore] = createStore(...)
```

这里返回的 `useCounter` 是 React hook。

而 hook 是 React 的机制，不属于 Vue，也不属于 Angular。

所以：

- `createStore` 不能直接拿去 Vue 用
- `useStore` 不能直接拿去 Angular 用

---

## 3. React 中如何使用

React 是 `nonestore` 当前支持最完整的框架。

因为它已经有专门的 React 适配层。

---

## 4. React 中推荐的使用方式

在 React 中，推荐直接使用：

- `createStore`
- `useStore`

### 示例

```tsx
import { createStore } from 'nonestore'

const [useCounter, counterStore] = createStore(
  { count: 0 },
  {
    computed: {
      doubled: (state) => state.count * 2,
    },
    actions: {
      increment: (state) => {
        state.count += 1
      },
      add: (state, amount: number) => {
        state.count += amount
      },
    },
  }
)

function App() {
  const count = useCounter((state) => state.count)
  const doubled = useCounter((state) => state.doubled)

  return (
    <div>
      <p>count: {count}</p>
      <p>doubled: {doubled}</p>
      <button onClick={() => counterStore.actions.increment()}>+1</button>
      <button onClick={() => counterStore.actions.add(5)}>+5</button>
    </div>
  )
}

export default App
```

### React 中的优势

React 中用 `nonestore` 有几个明显优点：

1. 已经有现成适配层
2. 组件里读取状态很直观
3. selector 用法清晰
4. 组件更新逻辑已经封装好

也就是说：

> React 是当前最适合使用 `nonestore` 的框架。

---

## 5. 在 Vue 中能不能用

可以，但不能直接照搬 React 写法。

也就是说：

- 可以使用 `create`
- 不能直接使用 `createStore`
- 不能直接使用 React 的 `useStore`

---

## 6. Vue 中应该怎么理解 nonestore

在 Vue 中，`nonestore` 更像是一个外部状态仓库。

Vue 自己有响应式系统：

- `ref`
- `reactive`
- `computed`
- `watchEffect`

而 `nonestore` 则提供：

- 状态存储
- 统一 action
- 订阅更新

所以在 Vue 中使用它的思路应该是：

1. 用 `create()` 创建框架无关的 store
2. 用 Vue 的 `ref` 或 `shallowRef` 保存当前状态快照
3. 用 `store.subscribe()` 监听变化
4. 变化后把最新状态同步回 Vue 响应式变量

也就是说：

> 在 Vue 中不是直接用 `nonestore` 替代 Vue 的响应式系统，而是把 `nonestore` 作为一个外部状态源，再桥接到 Vue。

---

## 7. Vue 的基础用法示例

### 第一步：创建 store

`stores/counter.ts`

```ts
import { create } from 'nonestore'

export const counterStore = create(
  { count: 0 },
  {
    computed: {
      doubled: (state) => state.count * 2,
    },
    actions: {
      increment: (state) => {
        state.count += 1
      },
      add: (state, amount: number) => {
        state.count += amount
      },
    },
  }
)
```

### 第二步：在 Vue 组件中桥接

```vue
<script setup lang="ts">
import { onMounted, onUnmounted, ref } from 'vue'
import { counterStore } from './stores/counter'

const snapshot = ref(counterStore.getState())

let unsubscribe: (() => void) | null = null

onMounted(() => {
  unsubscribe = counterStore.subscribe((state) => {
    snapshot.value = state
  })
})

onUnmounted(() => {
  unsubscribe?.()
})
</script>

<template>
  <div>
    <p>count: {{ snapshot.count }}</p>
    <p>doubled: {{ snapshot.doubled }}</p>
    <button @click="counterStore.actions.increment()">+1</button>
    <button @click="counterStore.actions.add(5)">+5</button>
  </div>
</template>
```

### 这段代码的核心逻辑

- `counterStore.getState()` 获取当前快照
- `counterStore.subscribe(...)` 监听状态变化
- 每次变化时，把新状态赋值给 `snapshot.value`
- Vue 模板通过 `snapshot` 自动更新

---

## 8. Vue 中更推荐的做法：封装一个组合式函数

在实际项目里，不建议每个 Vue 组件都手写一遍订阅逻辑。

更好的方式是封装一个组合式函数，比如 `useNonestore`。

### 示例

`composables/useNonestore.ts`

```ts
import { onMounted, onUnmounted, ref } from 'vue'

export function useNonestore<T extends { getState: () => any; subscribe: (listener: (state: any) => void) => () => void }>(store: T) {
  const state = ref(store.getState())
  let unsubscribe: (() => void) | null = null

  onMounted(() => {
    unsubscribe = store.subscribe((nextState) => {
      state.value = nextState
    })
  })

  onUnmounted(() => {
    unsubscribe?.()
  })

  return state
}
```

然后在组件里：

```vue
<script setup lang="ts">
import { counterStore } from './stores/counter'
import { useNonestore } from './composables/useNonestore'

const counter = useNonestore(counterStore)
</script>

<template>
  <div>
    <p>{{ counter.count }}</p>
    <p>{{ counter.doubled }}</p>
    <button @click="counterStore.actions.increment()">+1</button>
  </div>
</template>
```

这样会更接近 React 里“hook + store”的使用体验。

---

## 9. Vue 中使用 nonestore 的优点

1. 可以复用 `nonestore` 核心层逻辑
2. 如果你同时维护 React 和 Vue 项目，核心状态逻辑可以共享思路
3. action / computed 的结构比较清晰
4. 适合把业务规则集中管理

---

## 10. Vue 中使用 nonestore 的不足

1. 没有现成官方 Vue 适配层
2. 需要自己封组合式函数
3. 与 Vue 自带响应式相比，额外多一层桥接
4. 如果项目本来就完全 Vue 化，很多时候 Pinia 会更自然

所以结论是：

> 在 Vue 中可以用，但更多是“能适配”，不一定是“最佳原生方案”。

---

## 11. 在 Angular 中能不能用

可以，也同样建议只使用核心层 `create`。

也就是说：

- 可以用 `create`
- 不用 `createStore`
- 不用 React 的 hook

Angular 本身有自己的组件通信与响应系统，所以使用方式也需要桥接。

---

## 12. Angular 中应该怎么理解 nonestore

在 Angular 中，`nonestore` 更适合放在 service 层中。

可以把它理解成：

- `nonestore` 负责管理业务状态
- Angular service 负责向组件暴露状态和操作方法
- Angular 组件负责展示与交互

这是一种比较自然的组织方式。

---

## 13. Angular 的基础使用示例

### 第一步：创建核心 store

`counter.store.ts`

```ts
import { create } from 'nonestore'

export const counterStore = create(
  { count: 0 },
  {
    computed: {
      doubled: (state) => state.count * 2,
    },
    actions: {
      increment: (state) => {
        state.count += 1
      },
      add: (state, amount: number) => {
        state.count += amount
      },
    },
  }
)
```

### 第二步：放进 Angular service

`counter.service.ts`

```ts
import { Injectable } from '@angular/core'
import { counterStore } from './counter.store'

@Injectable({ providedIn: 'root' })
export class CounterService {
  store = counterStore

  getState() {
    return this.store.getState()
  }

  increment() {
    this.store.actions.increment()
  }

  add(amount: number) {
    this.store.actions.add(amount)
  }

  subscribe(listener: (state: ReturnType<typeof counterStore.getState>) => void) {
    return this.store.subscribe(listener)
  }
}
```

### 第三步：在组件中使用

`counter.component.ts`

```ts
import { Component, OnDestroy, OnInit } from '@angular/core'
import { CounterService } from './counter.service'

@Component({
  selector: 'app-counter',
  template: `
    <p>count: {{ count }}</p>
    <p>doubled: {{ doubled }}</p>
    <button (click)="increment()">+1</button>
    <button (click)="addFive()">+5</button>
  `,
})
export class CounterComponent implements OnInit, OnDestroy {
  count = 0
  doubled = 0
  private unsubscribe: (() => void) | null = null

  constructor(private counterService: CounterService) {}

  ngOnInit() {
    const snapshot = this.counterService.getState()
    this.count = snapshot.count
    this.doubled = snapshot.doubled

    this.unsubscribe = this.counterService.subscribe((state) => {
      this.count = state.count
      this.doubled = state.doubled
    })
  }

  ngOnDestroy() {
    this.unsubscribe?.()
  }

  increment() {
    this.counterService.increment()
  }

  addFive() {
    this.counterService.add(5)
  }
}
```

---

## 14. Angular 中更推荐的做法

在实际 Angular 项目里，建议：

1. 把 `nonestore` 放进 service
2. 在 service 中暴露统一方法
3. 组件只通过 service 操作状态

这样更符合 Angular 的习惯：

- 状态放 service
- UI 放 component
- 逻辑保持可测试

---

## 15. Angular 中使用 nonestore 的优点

1. 核心状态逻辑可以独立于组件
2. service 层组织方式比较自然
3. action 模型清晰，适合管理业务状态
4. 如果团队本来就喜欢状态集中，这种写法容易理解

---

## 16. Angular 中使用 nonestore 的不足

1. 没有现成 Angular 官方适配层
2. 需要自己在 service 中做桥接
3. 不如 Angular 自己的 signal / RxJS 生态那样原生
4. 在纯 Angular 项目里，不一定是最佳第一选择

所以结论也是：

> 在 Angular 中能用，但更多属于“可集成”，而不是“最原生方案”。

---

## 17. React、Vue、Angular 的差别总结

可以用一张表来理解：

| 框架 | 是否可用 | 推荐入口 | 是否有现成适配层 | 使用体验 |
| ---- | -------- | -------- | ---------------- | -------- |
| React | 可以 | `createStore` | 有 | 最完整 |
| Vue | 可以 | `create` | 没有 | 需要自己桥接 |
| Angular | 可以 | `create` | 没有 | 需要自己桥接 |

再换一种更直白的说法：

- React：开箱即用
- Vue：能用，但要自己封一层
- Angular：能用，但也要自己封一层

---

## 18. 如果你想让 nonestore 真正支持多框架，应该怎么做

如果后续想把 `nonestore` 做成真正的跨框架状态库，建议结构明确拆分成：

```txt
src/
├─ core.ts
├─ react.ts
├─ vue.ts
├─ angular.ts
└─ index.ts
```

### 建议思路

- `core.ts`：只保留纯状态容器逻辑
- `react.ts`：提供 React hook 绑定
- `vue.ts`：提供 Vue composable 绑定
- `angular.ts`：提供 Angular service / signal 绑定工具

这样做的好处：

1. 核心逻辑完全复用
2. 每个框架都有自己的适配层
3. 文档会更清晰
4. 用户更容易理解自己该导入什么

例如未来可能变成：

- React：`import { createStore } from 'nonestore/react'`
- Vue：`import { useNonestore } from 'nonestore/vue'`
- Angular：`import { createAngularStore } from 'nonestore/angular'`

这会比现在更专业。

---

## 19. 最终建议

### 如果你现在只做 React

直接用 `createStore`，这是当前最完整方案。

### 如果你想在 Vue 中试用

使用 `create`，再自己封一个 `useNonestore` 组合式函数。

### 如果你想在 Angular 中试用

使用 `create`，再放到 service 中做桥接。

### 如果你想做真正跨框架库

下一步应该补 Vue 和 Angular 的官方适配层，而不是只停留在 React hook 层。

---

## 20. 一句话总结

`nonestore` 现在的真实状态是：

> 核心层可以跨 React、Vue、Angular 使用，但真正开箱即用的适配目前只有 React；Vue 和 Angular 需要你自己做一层框架绑定。
