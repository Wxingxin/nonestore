# nonestore 课堂练习题与参考答案

## 1. 使用说明

这份练习题是配合 `nonestore` 教学使用的。

题目分成四类：

1. 基础题
2. 进阶题
3. 排错题
4. 思考题

建议课堂使用顺序：

1. 先做基础题，熟悉 `createStore`、`actions`、`computed`
2. 再做进阶题，理解 selector、persist、middleware
3. 再做排错题，训练学生发现问题和修复问题的能力
4. 最后做思考题，训练状态设计能力

## 2. 基础题

## 题目 1：创建最简单的计数器 store

### 要求

使用 `createStore` 创建一个计数器 store，要求：

- 初始状态是 `{ count: 0 }`
- 有一个 `increment` action
- 点击按钮后，页面显示的数字加 1

### 学生目标

理解：

- `createStore` 的基本结构
- `actions` 的作用
- 如何在 React 中读取状态

### 参考答案

```tsx
import { createStore } from 'nonestore'

const [useCounter, counterStore] = createStore(
  { count: 0 },
  {
    actions: {
      increment: (state) => {
        state.count += 1
      },
    },
  }
)

function App() {
  const count = useCounter((state) => state.count)

  return (
    <div>
      <p>count: {count}</p>
      <button onClick={() => counterStore.actions.increment()}>+1</button>
    </div>
  )
}

export default App
```

## 题目 2：添加带参数的 action

### 要求

在上一个题目的基础上，增加一个 `add` 方法，要求：

- 能传入数字参数
- 点击按钮后一次加 5

### 学生目标

理解：

- `actions` 不只是固定动作，还可以接收参数

### 参考答案

```tsx
import { createStore } from 'nonestore'

const [useCounter, counterStore] = createStore(
  { count: 0 },
  {
    actions: {
      increment: (state) => {
        state.count += 1
      },
      add: (state, amount: number) => {
        state.count += amount
      },
    },
  }
)

function App() {
  const count = useCounter((state) => state.count)

  return (
    <div>
      <p>count: {count}</p>
      <button onClick={() => counterStore.actions.increment()}>+1</button>
      <button onClick={() => counterStore.actions.add(5)}>+5</button>
    </div>
  )
}

export default App
```

## 题目 3：使用 computed 计算双倍值

### 要求

在计数器的基础上增加：

- `computed.doubled`
- 页面同时显示 `count` 和 `doubled`

### 学生目标

理解：

- `computed` 是派生值
- 派生值不需要手动存储

### 参考答案

```tsx
import { createStore } from 'nonestore'

const [useCounter, counterStore] = createStore(
  { count: 0 },
  {
    computed: {
      doubled: (state) => state.count * 2,
    },
    actions: {
      increment: (state) => {
        state.count += 1
      },
    },
  }
)

function App() {
  const count = useCounter((state) => state.count)
  const doubled = useCounter((state) => state.doubled)

  return (
    <div>
      <p>count: {count}</p>
      <p>doubled: {doubled}</p>
      <button onClick={() => counterStore.actions.increment()}>+1</button>
    </div>
  )
}

export default App
```

## 题目 4：年龄管理案例

### 要求

创建一个 `age` store，要求：

- 初始年龄是 `18`
- 有 `addone` 和 `deleteone` 两个 actions
- 页面可以显示年龄并控制加减

### 学生目标

理解：

- `actions` 可以定义多个方法
- 修改状态时统一通过 `store.actions.xxx()` 调用

### 参考答案

```tsx
import { createStore } from 'nonestore'

const [useAgeStore, ageStore] = createStore(
  { age: 18 },
  {
    actions: {
      addone: (state) => {
        state.age += 1
      },
      deleteone: (state) => {
        state.age -= 1
      },
    },
  }
)

function App() {
  const age = useAgeStore((state) => state.age)

  return (
    <div>
      <p>age: {age}</p>
      <button onClick={() => ageStore.actions.addone()}>Add One</button>
      <button onClick={() => ageStore.actions.deleteone()}>Delete One</button>
    </div>
  )
}

export default App
```

## 3. 进阶题

## 题目 5：用户信息 store

### 要求

创建一个用户信息 store，要求：

- 初始状态：`name: 'Alice'`, `age: 17`
- `computed.isAdult`
- `computed.profileText`
- `actions.setName`
- `actions.growUp`

页面展示：

- 用户名称与年龄
- 是否成年
- 按钮点击后改名和年龄加一

### 参考答案

```tsx
import { createStore } from 'nonestore'

const [useUserStore, userStore] = createStore(
  {
    name: 'Alice',
    age: 17,
  },
  {
    computed: {
      isAdult: (state) => state.age >= 18,
      profileText: (state) => `${state.name} (${state.age})`,
    },
    actions: {
      setName: (state, name: string) => {
        state.name = name
      },
      growUp: (state) => {
        state.age += 1
      },
    },
  }
)

function App() {
  const profileText = useUserStore((state) => state.profileText)
  const isAdult = useUserStore((state) => state.isAdult)

  return (
    <div>
      <p>{profileText}</p>
      <p>{isAdult ? '成年人' : '未成年人'}</p>
      <button onClick={() => userStore.actions.setName('Bob')}>改名</button>
      <button onClick={() => userStore.actions.growUp()}>长大一岁</button>
    </div>
  )
}

export default App
```

## 题目 6：使用 persist 保存主题

### 要求

创建一个设置 store，要求：

- 状态包含 `theme` 和 `language`
- 刷新页面后保留值
- 使用 `persist`

### 学生目标

理解：

- `persist` 是什么
- `key` 和 `pick` 的作用

### 参考答案

```tsx
import { createStore } from 'nonestore'

const [useSettingsStore, settingsStore] = createStore(
  {
    theme: 'light',
    language: 'zh-CN',
    tempText: '',
  },
  {
    persist: {
      key: 'settings-store',
      pick: ['theme', 'language'],
    },
    actions: {
      setTheme: (state, theme: string) => {
        state.theme = theme
      },
      setLanguage: (state, language: string) => {
        state.language = language
      },
    },
  }
)

function App() {
  const theme = useSettingsStore((state) => state.theme)
  const language = useSettingsStore((state) => state.language)

  return (
    <div>
      <p>theme: {theme}</p>
      <p>language: {language}</p>
      <button onClick={() => settingsStore.actions.setTheme('dark')}>dark</button>
      <button onClick={() => settingsStore.actions.setLanguage('en-US')}>english</button>
    </div>
  )
}

export default App
```

## 题目 7：使用 logger 中间件观察状态变化

### 要求

给计数器 store 添加 `logger` middleware，观察控制台输出。

### 学生目标

理解：

- middleware 的作用
- 如何在状态更新前后做额外工作

### 参考答案

```tsx
import { createStore, logger } from 'nonestore'

const [useCounter, counterStore] = createStore(
  { count: 0 },
  {
    middlewares: [logger({ collapsed: true, diff: true })],
    actions: {
      increment: (state) => {
        state.count += 1
      },
    },
  }
)

function App() {
  const count = useCounter((state) => state.count)

  return (
    <div>
      <p>{count}</p>

    </div>
  )
}

export default App
```

## 4. 排错题

## 题目 8：下面这段代码哪里错了

```tsx
const [useNameAge, nameageStore] = createStore(
  { age: 18 },
  {
    add: {
      addone: (state) => {
        state.age += 1
      },
    },
  }
)
```

### 正确答案

错误在于：

- 把 action 写在了 `add` 下面
- 正确位置应该是 `actions`

正确写法：

```tsx
const [useNameAge, nameageStore] = createStore(
  { age: 18 },
  {
    actions: {
      addone: (state) => {
        state.age += 1
      },
    },
  }
)
```

## 题目 9：下面这段代码为什么会报错

```tsx
<button onClick={() => nameageStore.add.addone()}>Add One</button>
```

### 正确答案

错误原因：

- `createStore` 返回的 store 实例里，action 在 `store.actions` 下面
- 不能写 `store.add.addone()`

正确写法：

```tsx
<button onClick={() => nameageStore.actions.addone()}>Add One</button>
```

## 题目 10：下面这段代码为什么会有红线

```tsx
computed: {
  doubled: (state) => state.count * 2,
},
actions: {
  increment: (state) => {
    state.count += 1
  },
}
```

如果 VS Code 提示：

```ts
Parameter 'state' implicitly has an 'any' type.
```

### 正确答案

这通常是库的类型推导设计问题，不一定是业务代码写错。

如果库类型已修复，`state` 应该能自动推导。

如果还没修复，可以临时手动标类型：

```tsx
computed: {
  doubled: (state: { count: number }) => state.count * 2,
},
actions: {
  increment: (state: { count: number }) => {
    state.count += 1
  },
}
```

## 题目 11：下面这段代码语法错误在哪里

```tsx
actions: {
  add: (state) => {
    state.count += 1
  }
  deleteone: (state) => {
    state.count -= 1
  }
}
```

### 正确答案

`add` 后面少了逗号。

正确写法：

```tsx
actions: {
  add: (state) => {
    state.count += 1
  },
  deleteone: (state) => {
    state.count -= 1
  },
}
```

## 5. 思考题

## 题目 12：哪些数据适合放进 computed

请思考下面哪些更适合放进 `computed`：

- `count`
- `price`
- `qty`
- `total = price * qty`
- `isAdult = age >= 18`

### 参考答案

更适合放进 `computed` 的是：

- `total`
- `isAdult`

因为它们都可以通过已有状态推导出来，不需要重复存储。

---

## 题目 13：哪些状态不一定要放进全局 store

请思考：

- 输入框是否聚焦
- 当前 tab 是否打开
- 一个 modal 是否可见
- 用户登录信息
- 购物车商品总数

### 参考答案

通常不一定要放进全局 store 的：

- 输入框是否聚焦
- 当前 tab 是否打开
- 一个 modal 是否可见

这些更适合组件本地 `useState`。

更适合全局或共享 store 的：

- 用户登录信息
- 购物车商品总数

---

## 题目 14：为什么不应该重复存储派生值

### 参考答案

因为重复存储容易带来数据不一致。

例如：

- `price = 10`
- `qty = 2`
- `total = 20`

如果后来改了 `qty = 3`，但忘了同步修改 `total`，数据就会错。

所以像 `total` 这种值，更适合放进 `computed`。

## 6. 综合题

## 题目 15：做一个购物车 store

### 要求

设计一个购物车 store，要求至少有：

- 商品列表 `items`
- `computed.total`
- `computed.itemCount`
- `actions.addItem`
- `actions.removeItem`
- `actions.clearCart`

### 参考答案

```tsx
import { createStore } from 'nonestore'

type CartItem = {
  id: number
  name: string
  price: number
  qty: number
}

const [useCartStore, cartStore] = createStore(
  {
    items: [] as CartItem[],
  },
  {
    computed: {
      total: (state) =>
        state.items.reduce((sum, item) => sum + item.price * item.qty, 0),
      itemCount: (state) =>
        state.items.reduce((sum, item) => sum + item.qty, 0),
    },
    actions: {
      addItem: (state, item: CartItem) => {
        const existing = state.items.find((it) => it.id === item.id)
        if (existing) {
          existing.qty += item.qty
        } else {
          state.items.push(item)
        }
      },
      removeItem: (state, id: number) => {
        state.items = state.items.filter((item) => item.id !== id)
      },
      clearCart: (state) => {
        state.items = []
      },
    },
  }
)

function App() {
  const items = useCartStore((state) => state.items)
  const total = useCartStore((state) => state.total)
  const itemCount = useCartStore((state) => state.itemCount)

  return (
    <div>
      <p>items: {itemCount}</p>
      <p>total: {total}</p>
      <button
        onClick={() =>
          cartStore.actions.addItem({ id: 1, name: 'Book', price: 30, qty: 1 })
        }
      >
        Add Book
      </button>
      <button onClick={() => cartStore.actions.clearCart()}>Clear</button>

      <ul>
        {items.map((item) => (
          <li key={item.id}>
            {item.name} - {item.qty} x {item.price}
          </li>
        ))}
      </ul>
    </div>
  )
}

export default App
```

## 7. 教师使用建议

如果你拿这份题目给学生练，建议这样安排：

### 第 1 轮

先做题目 1 到题目 4。

目标：

- 让学生会写最基本的 store
- 知道 `actions` 和 `computed` 的区别

做题目 5 到题目 7。

目标：

- 让学生理解业务状态设计
- 理解 `persist` 和 `middleware`

### 第 3 轮

做题目 8 到题目 11。

目标：

- 训练学生自己发现错误
- 强化对 API 结构的理解

### 第 4 轮

做题目 12 到题目 15。

目标：

- 让学生从“会写代码”升级为“会设计状态”

如果学生完成这套练习，理想情况下应该掌握：

- 什么是共享状态
- 如何用 `createStore` 创建 store
